====================
STAGE 1 — RUNTIME ALIGNMENT OF VERSION A
====================

Fix the concrete runtime-shape drifts in Version A.

Required:
1. `Subscribe to Session Requested`
   - remove `onBehalfOf` from low-level subscribe helpers
   - keep helper runtime-correct
   - if legacy call-shape compatibility is desired, prefer explicit docs over silently accepting unsupported fields

2. `grantSessionSubscriptionOnResult`
   - mark unsupported / deferred
   - do not materialize it into documents
   - update tests/docs accordingly

3. `Common/Named Event`
   - adopt the real public type from `@blue-repository/types@0.21.0`
   - no fallback mapping to `Conversation/Event`
   - event fields must stay on root, no fake `payload` wrapper

4. `MyOsPermissions`
   - align `write(...)` semantics to runtime-confirmed `share`

5. `Start Worker Session Requested`
   - align the emitted envelope to the final runtime shape:
     - `onBehalfOf`
     - `document`
     - `channelBindings`
     - `initialMessages`
     - `capabilities`
   - remove or rename legacy wrapper fields if present

6. Linked-doc helper semantics
   - ensure `onLinkedDocGranted(...)` means concrete linked-document notification
   - separate linked-documents grant lifecycle helpers from linked-document notification helpers
   - add the missing lifecycle helpers / aliases if needed

7. Audit PayNote runtime semantics
   - keep participant-aware defaults if runtime-correct
   - keep deferred items deferred
   - do not regress known-correct flows

STAGE 2 — IMPORT HIGH-VALUE HARDENING FROM VERSION B
====================

Bring the strongest parts of Version B into Version A, but preserve Version A structure and ergonomics.

Import or recreate in Version A:
1. Strong public exports / utilities where missing
2. Strong parity + integration + canonical scenario tests
3. Final corrections around:
   - named events
   - linked-doc semantics
   - compatibility wrappers
4. Editing pipeline hardening:
   - `DocStructure`
   - `DocPatch`
   - `BlueChangeCompiler`

5. Final docs/deviation discipline
6. Public-repo hygiene:

   - no references to private repos in committed tests/docs

Important:
- Port tests and semantics from Version B
- Do NOT port Version B’s monolithic builder structure
- Prefer adapting the good ideas into Version A’s modular modules
